JAVA Maven IDEA三者环境配置

JDK、JRE环境配置
1、下载JDK文件并安装
访问网站：https://repo.huaweicloud.com/java/jdk/8u151-b12/
选择Window系统下的64位jdk

然后安装到自己指定位置也可以是默认位置。

2、配置JDK环境
找到并打开电脑环境变量，在系统变量中，新建变量：JAVA_HOME， 变量值：jdk的根目录

找到系统变量中Path变量，并新建两个：%JAVA_HOME%’\bin和%JAVA_HOME%\jre\bin保存即可

3、验证环境配置是否成功
在cmd命令窗口中输入java，java -verion， javac
无报错则成功配置java环境

Maven环境配置
1、下载maven文件并安装
访问网站：https://mirrors.tuna.tsinghua.edu.cn/apache/maven/maven-3/3.6.3/binaries/apache-maven-3.6.3-bin.zip
下载压缩包，解压到指定位置建议和JDK包放一起方便管理

2、maven环境配置
打开环境变量，添加系统变量，在系统变量中，新建变量MAVEN_HOME，变量值为：maven解压文件的根目录

在系统变量Path中，添加%MAVEN_HOME%\bin

3、验证maven环境配置是否成功
打开cmd控制台，输入命令：mvn -v

4、修改相关包的下载源与仓库位置
打开maven解压文件下的conf文件，找到配置文件Settings.xml文件。

更改Anaconda中Jupyter文件默认路径

1、找到配置文件

  • 菜单中打开Anaconda Prompt
  • 输入命令jupyter notebook –generate-config
  • 根据上面命令输出的路径找到配置文件

2、更改配置

  • 找到#c.NotebookApp.note_dir = ‘’，去掉该行前面的“**#”；再打算存放文件的位置新建一个文件夹（注意：最好是英文的）**，然后将新的路径设置在单引号中，保存配置文件。
  • 在开始菜单找到“Jupyter Notebook”快捷键，鼠标右击 -> 更多 -> 打开文件位置。
  • 找到对应的“Jupyter Notebook”快捷图标，鼠标右击 -> 属性 -> 目标， 去掉后面“**%USERPROFILE/%”（很重要**），然后点击 “应用”。
  • 重新启动Jupyter 即可

深拷贝与浅拷贝的区别详解

一、数据类型
数据分为基本数据类型（String，Number，Boolean，Null，Undefined，Symbol）和对象数据类型
1、基本数据类型的特点：直接存储在栈（Stack）中的数据
2、引用数据类型的特点：存储的是该对象在栈中的引用，真是数据存放在堆内存里


引用数据类型在栈中存储了指针，该指针指向堆中该实体的起始地址。当解释器寻找引用值时，会首先检索其在栈中的地址，取得地址后从堆中获得实体。
![Alt text]( https://upload-images.jianshu.io/upload_images/13253432-74602a0f1cc0d432?imageMogr2/auto-orient/strip|imageView2/2/format/webp"optional title”)


二、浅拷贝与深拷贝
深拷贝是只针对与像Object这样的引用类型的。
深拷贝和浅拷贝的示意图大致如下：
![Alt text]( https://upload-images.jianshu.io/upload_images/13253432-aca35e0020358b3d?imageMogr2/auto-orient/strip|imageView2/2/w/310/format/webp"optional title”)
浅拷贝只复制指向某个对象的指针，而不复制对象本身，新旧对象还是共享同一块内存。但深拷贝会另外创造一个一模一样的对象，新对象跟原对象不共享内存，修改新对象不会改到原对象。


三、赋值和浅拷贝的区别
赋值：当我们把一个对象赋值给一个新的变量时，赋的其实是该对象的在栈中的地址，而不是堆中的数据。也就是两个对象指向的是同一个存储空间，无论哪个对象发生改变，其实都是改变的存储空间的内容，因此，两个对象是联动的。

浅拷贝：浅拷贝是按位拷贝对象，它会创建一个新对象，这个对象有着原始对象属性值的一份精确拷贝。如果属性是基本类型，拷贝的就是基本类型的值；如果属性是内存地址（引用类型），拷贝的就是内存地址 ，因此如果其中一个对象改变了这个地址，就会影响到另一个对象。即默认拷贝构造函数只是对对象进行浅拷贝复制(逐个成员依次拷贝)，即只复制对象空间而不复制资源。
